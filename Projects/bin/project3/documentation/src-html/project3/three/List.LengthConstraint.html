146        /**
147         * CreateList is a general list constructor. It is customized in a
148         * subclass by possibly overridding the methods:
149         * 
150         * <ul>
151         *     <li>
152         *     createNode(Lex) -- Should be overridden in every specialization.
153         *     By defining it to create only instances of <b>X</b> (<b>X</b> must
154         *     be a specialization of <b>Node</b>), then we can restrict the
155         *     specialization of <b>List</b> to be a list of <b>X</b>.
156         *     </li>
157         *
158         *     <li>
159         *     getSeparator(Lex) -- When parsing a list, some lists are separated
160         *     by commas and some are not.  This method defines whether it is comma
161         *     separated or not.
162         *     </li>
163         *
164         *     <li>
165         *     isInFirstOfListElement(Lex) -- Different lists must start with
166         *     different kinds of tokens (e.g. some must start with an
167         *     <b>Identifier</b> and others may start with <b>Constant</b>s).
168         *     This method defines what kind of <b>Token</b> a list must start
169         *     with.
170         *     </li>
171         *
172         *     <li>
173         *     getLengthConstraint(Lex) -- Some lists are lists are lists of 0 or
174         *     more elements.  Other lists are lists of 1 or more elements.  This
175         *     is overridden to define which type of list we are woking with.
176         *     </li>
177         * </ul>
178         * <p>    
179         *     See their descriptions below.
180         * <p>    
181         * Normally, this code would go in the <i>List(Lex)</i> constructor, but,
182         * in the subclasses this code needs to be executed after doing some
183         * processing.  In a method we can do some processing and then call the
184         * same method in the superclass.  But, with a constructor we must call
185         * the constructor in the superclass first.  We are unable do any
186         * processing or checking first. So, we have the constructor call a method 
187         * method that does some processing and then calls <i>createList(Lex)</i>
188         * method.  In some sense, <i>createList(Lex)</i> is the constructor, for
189         * the class <b>List</b>.<p>
190         *
191         * <p>
192         * <table>
193         *   <tr>
194         *     <td valign=top>
195         *       <b>Precondition</b>:
196         *     </td>
197         *     <td valign=top>
198         *       lex &ne; null
199         *     </td>
200         *     <td valign=top>
201         *     </td>
202         *   </tr>
203         *   <tr>
204         *     <td valign=top>
205         *       <b>Postcondition</b>:
206         *     </td>
207         *     <td valign=top>
208         *         a list of Nodes is created according to the 4 methods
209         *               identified.
210         *     </td>
211         *     <td valign=top>
212         *     </td>
213         *   </tr>
214         * </table>
215         *
216         * @throws ParserException
217         *
218         * @param lex the lexical analyzer we will create the <b>List</b> from.j
219         * @see
220         *   <a href=
221         *      "../../srchtml/project3/List.html#LexConstructor"
222         *      >Source Code
223         *   </a>
224         */
225        public void createList(Lex lex)
226            throws ParserException
227        {
228            if(lex == null)throw new ParserException(
229                "ERROR in List::createList(Lex)\n" +
230                "    lex cannot be null");
231              
232            nodes = new ArrayList<Node>();
233            boolean anotherNode = checkForFirstMemberOfTheList(lex);
234            while(anotherNode){
235                nodes.add(createNode(lex));
236                anotherNode = checkForAnotherMemberOfTheList(lex);
237            };
238        }

265         *     </td>

278        public String toString()
279        {
280            StringBuffer result = new StringBuffer();
281            for(int i = 0; i < nodes.size(); i++){
282                result.append(nodes.get(i).toString());
283                if(i < nodes.size() - 1){
284                    Token token = getSeparator();
285                    result.append(token == null ? "" : token.getValue());
286                };
287            };
288            return result.toString();
289        }

412        /**
413         * Checks to see if there is a first member of the list.  Lists come in two
414         * forms: list of length 1 or more and lists of length 0 or more.  The most
415         * common is a list of 1 or more members.  If a list has 1 or more members,
416         * then this method should be a method that is constant and always returns
417         * true.  This is the default implementation of this method.  
418         *
419         * <p>
420         * <table>
421         *   <tr>
422         *     <td valign=top>
423         *       <b>Precondition</b>:
424         *     </td>
425         *     <td valign=top>
426         *       lex &ne; null
427         *     </td>
428         *     <td valign=top>
429         *     </td>
430         *   </tr>
431         *   <tr>
432         *     <td valign=top>
433         *       <b>Postcondition</b>:
434         *     </td>
435         *     <td valign=top>
436         *       getListConstraint = ONE_OR_MORE &rArr; result = true
437         *     </td>
438         *     <td valign=top>
439         *       AND
440         *     </td>
441         *   </tr>
442         *   <tr>
443         *     <td valign=top>
444         *       &nbsp;
445         *     </td>
446         *     <td valign=top>
447         *       getListConstraint = ZERO_OR_MORE &rArr;
448         *           result = checkForAnotherMemberOfTheList(lex)
449         *     </td>
450         *     <td valign=top>
451         *     </td>
452         *   </tr>
453         * </table>
454         *
455         * @throws ParserException
456         *
457         * @param lex the lexical analyzer we will check to for a first member of
458         *        the <b>List</b>
459         *
460         * @return whether there must be a first member of the list.
461         * @see
462         *   <a href=
463         *      "../../srchtml/project3/List.html#checkForFirstMemberOfTheList"
464         *      >Source Code
465         *   </a>
466         */
467        protected boolean checkForFirstMemberOfTheList(Lex lex)
468            throws ParserException
469        {
470            //The only method to call this method should be createList(Lex).  In
471            //that method we check to see if lex is null and, if so, throw an
472            //exception.  Thus, lex should never be null.
473            assert lex != null;
474    
475            boolean result = false;
476            switch(getLengthConstraint()){
477                case ONE_OR_MORE:
478                    result = true;
479                    break;
480                case ZERO_OR_MORE:
481                    result = isInFirstOfListElement(lex);
482                    break;
483            };
484            return result;
485        }

487        /**
488         * Determines whether the next token is of the proper Type.  We typically
489         * use this method to determine whether we are to continue parsing the
490         * elements of a list.  In most cases, we check to see if the next token
491         * is an <i>Identifier</i>.  Thus, this is the default behavior.  In another
492         * case we check to see if the token is an Argument.  In lists where there
493         * is a separator, we check for the separator, and, if it exists,
494         * consume it.
495         *
496         * <p>
497         * <table>
498         *   <tr>
499         *     <td valign=top>
500         *       <b>Precondition</b>:
501         *     </td>
502         *     <td valign=top>
503         *       none
504         *     </td>
505         *     <td valign=top>
506         *     </td>
507         *   </tr>
508         *   <tr>
509         *     <td valign=top>
510         *       <b>Postcondition</b>:
511         *     </td>
512         *     <td valign=top>
513         *       getSeparator() == null &rArr; result = isInFirstOfListElement(lex)
514         *     </td>
515         *     <td valign=top>
516         *       AND
517         *     </td>
518         *   </tr>
519         *   <tr>
520         *     <td valign=top>
521         *       &nbsp;
522         *     </td>
523         *     <td valign=top>
524         *       getSeparator() &ne; null &rArr;
525         *         (lex.front().getTokenType() == getSeparator().getTokenType() AND
526         *                        lex = lex'.popFront())
527         *     </td>
528         *     <td valign=top>
529         *     </td>
530         *   </tr>
531         * </table>
532         *
533         * @throws ParserException
534         *
535         * @param lex the lexical analyzer we will check to see if there is
536         *              another member of the <b>List</b>
537         *
538         * @return whether there is a nother member of the list.
539         * @see
540         *   <a href=
541         *      "../../srchtml/project3/List.html#checkForAnotherMemberOfTheList"
542         *      >Source Code
543         *   </a>
544         */
545        protected boolean checkForAnotherMemberOfTheList(Lex lex)
546            throws ParserException
547        {
548            //The only method to call this method should be createList(Lex).  In
549            //that method we check to see if lex is null and, if so, throw an
550            //exception.  Thus, lex should never be null.
551            assert lex != null;
552    
553            boolean result = false;
554            Token separator = getSeparator();
555            if(separator == null){ //this means there is no separator
556                result = isInFirstOfListElement(lex);
557            } else {
558                result = lex.front().getTokenType() == separator.getTokenType();
559                if(result){
560                    lex.popFront();
561                };
562            };
563            return result;
564        }

678        /**
679         * If we have a list of elements of type <b>T</b>, this determines if
680         * lex.front() is in the first of <b>T</b>.  Normally there is a set of
681         * token types associated with a the first of of type <b>T</b>, but, in
682         * this grammar the set only has a single type, usually, <b>Identifier</b>. 
683         * Thus, the default check is to check for an <b>Identifer</b>.  In
684         * <b>Predicate</b>s we override this method and check for <b>Argument</b>s.
685         *
686         * <p>
687         * <table>
688         *   <tr>
689         *     <td valign=top>
690         *       <b>Precondition</b>:
691         *     </td>
692         *     <td valign=top>
693         *       none
694         *     </td>
695         *     <td valign=top>
696         *     </td>
697         *   </tr>
698         *   <tr>
699         *     <td valign=top>
700         *       <b>Postcondition</b>:
701         *     </td>
702         *     <td valign=top>
703         *       result = lex.front() <i>instanceof</> <b>Identifier</b>
704         *     </td>
705         *     <td valign=top>
706         *     </td>
707         *   </tr>
708         * </table>
709         *
710         * @param lex we check lex[0] to see if it is of the appropriate type.
711         *
712         * @return wheter lex[0] is in the first of <i>T</i> for this
713         *         <b>List</b>.
714         * @see
715         *   <a href=
716         *      "../../srchtml/project3/List.html#isInFirstOfListElement"
717         *      >Source Code
718         *   </a>
719         */
720        protected boolean isInFirstOfListElement(Lex lex)
721        {
722            // The method checkForAnotherMemberOfTheList(Lex) is the only
723            // method that calls this method.  checkForAnotherMemberOfTheList(Lex)
724            // assumes, and we have proved, the the program as it currently stands
725            // will not allow lex to be null at this point.  Thus, we make the
726            // assertion.
727            assert lex != null ;
728    
729            return lex.front() instanceof Identifier;
730        }

732        /**
733         * If the list must have one or more members, then return ONE_OR_MORE, else
734         * return ZERO_OR_MORE.  By default we assume that a list must have one
735         * or more elements.  It is possible to have lists with other contraints
736         * on their lengths (e.g. 2 or more).  We have designed this so we
737         * can change the enumerated type and change this method to accomodate other
